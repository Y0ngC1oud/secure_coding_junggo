# Tiny Market (secure_coding_junggo)

Flask 기반 중고거래 플랫폼 과제. 회원가입/로그인, 상품 등록·조회·검색, 사용자 간 채팅(전체/1:1), 신고 및 자동 제재, 사용자 간 송금, 관리자 페이지를 제공한다.

## 기술 스택

- Python 3 + Flask
- SQLAlchemy (SQLite)
- Flask-Login (세션 인증), Flask-WTF (폼/CSRF), Flask-Limiter (요청 제한)
- Jinja2 서버사이드 템플릿

## 환경 설정

```bash
git clone https://github.com/Y0ngC1oud/secure_coding_junggo.git
cd secure_coding_junggo

python3 -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env
```

[`.env.example`](.env.example)에 바로 동작하는 기본값이 이미 채워져 있어서 **별도 수정 없이 그대로 써도 된다.** 운영 배포 시에는 `SECRET_KEY`와 `ADMIN_PASSWORD`를 직접 원하는 값으로 바꾸는 것을 권장한다(`.env` 파일만 수정하면 되며, 수정 후에는 서버를 재시작해야 반영된다).

## 실행 방법

```bash
PORT=5050 python run.py
```

`http://localhost:5050` 접속. 종료는 터미널에서 `Ctrl + C`.

> **macOS 사용자 주의**: macOS는 기본적으로 5000번 포트를 "AirPlay 수신 기능"이 사용 중이라 `PORT` 지정 없이 실행하면 접속 시 403 에러가 날 수 있다. 위처럼 5050 등 다른 포트를 지정하거나, 시스템 설정 > 일반 > AirDrop 및 Handoff에서 "AirPlay 수신 기능"을 꺼도 된다.

최초 실행 시 `instance/app.db` SQLite 파일과 테이블이 자동으로 생성된다. 데이터를 전부 초기화하려면 서버를 끄고 `rm instance/app.db` 실행 후 다시 켜면 된다.

## 관리자 계정 만들기

**방법 1 (권장) — `.env` 값 그대로 사용**

`.env.example`을 그대로 복사했다면 별도 설정 없이 아래 계정으로 바로 로그인하면 관리자다.

- 아이디: `admin`
- 비밀번호: `admin1234`

서버를 시작할 때마다 `.env`의 `ADMIN_USERNAME`/`ADMIN_PASSWORD` 값으로 해당 계정이 자동 생성되고 관리자 권한·비밀번호가 항상 동기화된다(이미 다른 비밀번호로 가입되어 있던 계정이어도 `.env` 값으로 덮어씀). 비밀번호를 바꾸고 싶으면 `.env`의 `ADMIN_PASSWORD`만 수정하고 서버를 재시작하면 된다.

**방법 2 — CLI 명령**

1. 먼저 웹 화면에서 일반 회원가입을 진행한다.
2. 아래 명령에서 `가입한아이디` 부분을 실제 아이디 문자열로 바꿔서 실행한다(꺾쇠괄호 `<`, `>`는 입력하지 않음). **반드시 프로젝트 폴더로 이동해 venv를 켠 상태**에서 실행해야 한다.

```bash
cd secure_coding_junggo
source .venv/bin/activate
flask --app run create-admin 가입한아이디
```

관리자로 로그인하면 상단 네비게이션에 "관리자" 메뉴가 나타나며 `/admin`에서 회원/상품/신고를 관리할 수 있다.

## 사용 방법

| 기능 | 위치 | 방법 |
|---|---|---|
| 회원가입/로그인 | 상단 메뉴 | 아이디·닉네임·비밀번호 입력 (신규 가입자는 잔액 0원으로 시작) |
| 상품 등록 | 상품 목록 → "+ 상품 등록" | 상품명·설명·가격·카테고리·사진(선택) |
| 상품 둘러보기 | 상품 목록 | 이름 클릭 시 상세 페이지 |
| 상품 검색 | 상품 목록 검색창 | 키워드/카테고리 필터, `/products/search`로 이동 |
| 전체 채팅 | 상품 목록 하단 | 로그인 후 메시지 입력 |
| 1:1 채팅 | 상품 상세 → "판매자와 채팅하기" | 또는 상단 "채팅" 메뉴에서 대화 목록 확인 |
| 구매하기 | 상품 상세 → "구매하기" | 잔액이 있어야 함. 구매 시 판매자에게 자동 송금 + 상품이 판매완료(sold) 처리됨 |
| 송금 | 상단 "송금" | 받는 사람 아이디/닉네임 + 금액 입력 |
| 거래 내역 | 송금 페이지 → "거래 내역 보기" | 보낸/받은 내역 확인 |
| 신고 | 상품 상세 → "상품 신고"/"판매자 신고" | 사유 작성, 3회 누적 시 자동 차단(상품)·휴면(사용자) |
| 마이페이지 | 상단 "마이페이지" | 소개글/비밀번호 수정, 잔액·내 상품 확인 |
| 관리자 | 관리자 계정 로그인 시 상단 "관리자" | 회원 관리(정지/활성화/**잔액 직접 설정**), 상품 관리(차단/삭제), 신고 관리(처리완료) |

> 회원가입 시 잔액은 0원으로 시작하므로, 송금·구매 기능을 테스트하려면 관리자 → 회원 관리에서 해당 계정 잔액을 먼저 설정해야 한다.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 회원가입/로그인/로그아웃 | 세션 기반 인증, 비밀번호 해시 저장, 로그인 브루트포스 방지 |
| 마이페이지 | 소개글 수정, 비밀번호 변경, 잔액/등록 상품 확인 |
| 상품 등록/조회/수정/삭제 | 이미지 업로드(확장자 검증 + 랜덤 파일명), 소유자만 수정/삭제 가능 |
| 상품 구매 | 원클릭 구매(송금 + 판매완료 처리를 하나의 트랜잭션으로 원자적 처리) |
| 상품 검색 | 별도 검색 페이지에서 키워드 + 카테고리 필터 |
| 채팅 | 전체 채팅, 사용자 간 1:1 채팅 |
| 신고 | 상품/사용자 신고, 사유 작성, 누적 시 자동 차단(상품)·휴면 처리(사용자) |
| 송금 | 사용자 간 잔액 기반 송금(아이디/닉네임 지정), 거래 내역 조회 |
| 공개 프로필 | 다른 사용자의 닉네임·소개글·등록 상품 열람 |
| 관리자 | 회원 정지/활성화/잔액 설정, 상품 활성화/차단/삭제, 신고 처리 |
